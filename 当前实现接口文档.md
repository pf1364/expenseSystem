# 当前实现接口文档

## 统一返回格式

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

异常参数返回 `code=400`，系统异常返回 `code=500`。

## 1. 报销单分页查询

`GET /api/reimbursements/page`

查询参数：

| 字段 | 说明 |
| --- | --- |
| pageNum/pageNo | 当前页 |
| pageSize | 每页条数 |
| reimNo | 报销单号，模糊查询 |
| title | 标题，模糊查询 |
| reason | 事由，模糊查询 |
| reimCompanyName | 费用归属公司，主表冗余字段模糊查询 |
| reimDepartmentName | 报销部门，精确查询 |
| reimburserKeyword | 报销人姓名或工号，精确查询 |
| reimburserName | 报销人姓名，精确查询 |
| reimburserNo | 报销人工号，精确查询 |
| businessTypeName | 业务类型，精确查询 |
| billStatus | 单据状态，精确查询 |

## 2. 报销单详情

`GET /api/reimbursements/{reimNo}`

返回主表、行程、每日补助明细、费用归属及分摊。

## 3. 创建草稿

`POST /api/reimbursements/draft`

草稿允许基础信息不完整。后端生成 `reimNo`，状态写为 `DRAFT/草稿`。

## 4. 创建并提交

`POST /api/reimbursements/submit`

提交时校验：

- 报销标题、报销人、报销部门、业务类型、出差事由必填。
- 至少一条完整行程。
- 每日补助实报金额不能超过城市标准。
- 分摊比例合计必须为 100%。
- 分摊金额合计必须等于报销总金额，允许 0.01 误差。

## 5. 更新报销单

`PUT /api/reimbursements/{reimNo}`

子表按 ID 增删改：

- 请求带 ID 且数据库存在：更新。
- 请求不带 ID：新增。
- 数据库存在但请求未带回：删除。

## 6. 提交已有草稿

`POST /api/reimbursements/{reimNo}/submit`

将已有草稿校验后改为 `SUBMITTED/已提交`。

## 7. 删除草稿

`DELETE /api/reimbursements/{reimNo}`

只允许删除 `DRAFT` 状态单据，同时删除行程、每日补助和分摊子表。

## 8. 作废单据

`POST /api/reimbursements/{reimNo}/void`

草稿不能作废，草稿应直接删除。已提交单据作废后状态为 `VOIDED/已作废`。

## 9. 生成每日补助

`POST /api/reimbursements/allowance-days/generate`

请求体：

```json
{
  "startCityCode": "10458",
  "startCityName": "武汉",
  "endCityCode": "10119",
  "endCityName": "北京",
  "startDate": "2026-04-13",
  "endDate": "2026-04-17"
}
```

返回每天一条补助明细。默认勾选餐补、交通补助、通讯补助，金额等于城市标准。前端可以改小或取消勾选，后端保存和提交时会再次校验。

## 10. 城市补助标准

`GET /api/city-allowances`

返回城市编码、城市名称、城市等级、餐补标准、交通补助标准、通讯补助标准。

## 11. 个人信息统计

`GET /api/statistics/personal`

查询参数：

| 字段 | 说明 |
| --- | --- |
| reimburserName | 报销人姓名 |
| reimburserNo | 报销人工号 |

返回：

- 报销单总数
- 草稿数量
- 已提交数量
- 累计报销金额
- 月度报销金额柱状图数据
- 费用归属公司占比图数据
- 最近报销单列表
