# 差旅报销单接口文档

## 1、报销单分页查询

### 接口说明

用于 5.1.1 报销单列表页面分页查询。查询条件可以全部为空，也可以按条件组合查询。

其中：

- 报销单号、标题、事由是模糊查询。
- 费用归属公司、报销部门、报销人、业务类型是精确查询。
- 当前报销单主表 `fk_reim_main.reim_company_names` 保存费用归属公司名称汇总，用于列表页展示和查询。权威分摊数据仍然是 `fk_reim_allocation`，创建/更新分摊时需要同步回写主表缓存字段。

### 请求地址

```http
GET /api/reimbursements/page
```

## 2、创建并提交报销单

### 接口说明

用于用户填写完整报销单后点击“提交”。前端传入完整单据数据，后端需要创建报销单主表、行程表、每日补助明细表、费用归属及分摊表，并将单据状态设置为已提交。

创建时 `reimNo` 可以为空，后端生成唯一报销单号。

### 请求地址

```http
POST /api/reimbursements/submit
```

### 请求体

请求体和创建草稿、更新接口共用同一套结构。

| 字段名 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| reimNo | String | N | 报销单号；创建时为空 |
| title | String | Y | 报销标题 |
| reimburserId | String | N | 报销人ID，当前前端写死，可为空 |
| reimburserNo | String | N | 报销人工号 |
| reimburserName | String | Y | 报销人姓名 |
| reimDepartmentId | String | N | 报销部门ID，当前前端写死，可为空 |
| reimDepartmentNo | String | N | 报销部门编号 |
| reimDepartmentName | String | Y | 报销部门名称 |
| reimCompanyNames | String | N | 费用归属公司名称汇总，可由后端根据分摊信息回写 |
| businessTypeId | String | N | 业务类型ID，当前前端写死，可为空 |
| businessTypeNo | String | N | 业务类型编号 |
| businessTypeName | String | Y | 业务类型名称 |
| reason | String | Y | 出差事由 |
| remark | String | N | 备注 |
| itineraries | Array | Y | 行程信息，可以有多条 |
| allocations | Array | Y | 费用归属及分摊信息 |

`itineraries` 字段：

| 字段名 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| id | Long | N | 行程ID，创建时为空 |
| travelerId | String | N | 出行人ID |
| travelerNo | String | N | 出行人工号 |
| travelerName | String | Y | 出行人姓名 |
| startCityCode | String | Y | 出发城市编码 |
| startCityName | String | Y | 出发城市名称 |
| endCityCode | String | Y | 到达城市编码 |
| endCityName | String | Y | 到达城市名称 |
| startDate | LocalDate | Y | 出发日期 |
| endDate | LocalDate | Y | 到达日期 |
| days | Integer | Y | 行程天数 |
| routeText | String | Y | 行程展示文本 |
| description | String | Y | 行程说明 |
| allowanceDays | Array | Y | 当前行程对应的每日补助明细 |

`allowanceDays` 字段：

| 字段名 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| id | Long | N | 每日补助明细ID，创建时为空 |
| allowanceDate | LocalDate | Y | 补助日期 |
| weekName | String | Y | 星期名称 |
| cityCode | String | Y | 补助城市编码 |
| cityName | String | Y | 补助城市名称 |
| cityLevel | Integer | Y | 城市等级：1一线，2二线，3三线 |
| mealStandard | BigDecimal | Y | 餐费补助标准金额 |
| mealSelected | Integer | Y | 是否选择餐费补助：1是，0否 |
| mealAmount | BigDecimal | Y | 实际餐费补助金额 |
| trafficStandard | BigDecimal | Y | 交通补助标准金额 |
| trafficSelected | Integer | Y | 是否选择交通补助：1是，0否 |
| trafficAmount | BigDecimal | Y | 实际交通补助金额 |
| communicationStandard | BigDecimal | Y | 通讯补助标准金额 |
| communicationSelected | Integer | Y | 是否选择通讯补助：1是，0否 |
| communicationAmount | BigDecimal | Y | 实际通讯补助金额 |
| dayAmount | BigDecimal | Y | 当日补助合计 |

`allocations` 字段：

| 字段名 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| id | Long | N | 分摊ID，创建时为空 |
| allocationOwnerType | String | Y | 分摊归属方类型：COMPANY公司，DEPARTMENT部门 |
| allocationOwnerId | String | N | 分摊归属方ID |
| allocationOwnerNo | String | N | 分摊归属方编号 |
| allocationOwnerName | String | Y | 分摊归属方名称 |
| businessId | String | N | 分摊业务ID |
| businessName | String | N | 分摊业务名称 |
| allocationRatio | BigDecimal | Y | 分摊比例，0到1 |
| allocationAmount | BigDecimal | Y | 分摊金额 |
| sortNo | Integer | Y | 排序号 |

### 后端处理逻辑

1. 生成唯一 `reimNo`。
2. 保存 `fk_reim_main`，状态为 `SUBMITTED`，状态名为 `已提交`。
3. 保存 `fk_reim_itinerary`。
4. 保存每个行程下的 `fk_reim_allowance_day`。
5. 保存 `fk_reim_allocation`。
6. 汇总每日补助明细，回写主表：
   - `allowance_amount`
   - `meal_amount`
   - `traffic_amount`
   - `communication_amount`
7. 汇总分摊表中 `allocation_owner_type = COMPANY` 的公司名称，回写主表 `reim_company_names`。
8. 以上操作需要放在同一个事务中。

### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "reimNo": "RCBX20260601001",
    "billStatus": "SUBMITTED",
    "billStatusName": "已提交"
  }
}
```

## 3、创建草稿

### 接口说明

用于用户填写部分或全部信息后保存草稿。请求体与“创建并提交报销单”一致，区别是后端将单据状态设置为草稿。

草稿在前端列表或详情中应展示为可修改。

### 请求地址

```http
POST /api/reimbursements/draft
```

### 后端处理逻辑

1. 创建时 `reimNo` 可以为空，后端生成唯一报销单号。
2. 保存主表、行程、每日补助明细、分摊信息。
3. 单据状态设置为 `DRAFT`，状态名为 `草稿`。
4. 草稿允许字段不完整，但后端仍应进行基础类型、长度、金额不能为负等校验。

### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "reimNo": "RCBX20260601002",
    "billStatus": "DRAFT",
    "billStatusName": "草稿"
  }
}
```

## 4、更新报销单

### 接口说明

用于修改已经存在的报销单，主要场景是修改草稿。前端需要传入 `reimNo`，请求体仍然使用完整单据结构。

更新时建议采用“整单覆盖保存”的方式：先按 `reimNo` 找到主表，再更新主表，并重建或同步行程、每日补助明细、分摊信息。

### 请求地址

```http
PUT /api/reimbursements/{reimNo}
```

### 路径参数

| 参数名 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| reimNo | String | Y | 报销单号 |

### 后端处理逻辑

1. 根据 `reimNo` 查询报销单。
2. 只有草稿状态建议允许修改；如果已提交或审批中，后续需要按业务规则限制。
3. 更新主表基础信息。
4. 更新行程信息。
5. 更新每日补助明细。
6. 更新费用归属及分摊信息。
7. 重新汇总金额，并回写主表金额字段。
8. 重新汇总费用归属公司名称，并回写 `reim_company_names`。
9. 全流程放在同一个事务中。

### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "reimNo": "RCBX20260601002",
    "billStatus": "DRAFT",
    "billStatusName": "草稿"
  }
}
```

### 请求参数

| 参数名 | 类型 | 是否必填 | 查询方式 | 说明 |
| --- | --- | --- | --- | --- |
| pageNum | Integer | N |  | 当前页，默认 1 |
| pageSize | Integer | N |  | 每页条数，默认 10 |
| reimNo | String | N | 模糊 | 报销单号，例如输入 `100` 可匹配包含 `100` 的单号 |
| title | String | N | 模糊 | 报销标题 |
| reason | String | N | 模糊 | 报销事由 |
| reimCompanyName | String | N | 精确 | 费用归属公司名称；查询主表缓存字段 `reim_company_names` |
| reimDepartmentName | String | N | 精确 | 报销部门名称；对应主表 `reim_department_name` |
| reimburserKeyword | String | N | 精确 | 报销人姓名或工号；匹配主表 `reimburser_name` 或 `reimburser_no` |
| businessTypeName | String | N | 精确 | 业务类型名称；对应主表 `business_type_name` |

### 查询逻辑

1. 主查询表是 `fk_reim_main`。
2. `reimNo`、`title`、`reason` 使用 `LIKE` 查询。
3. `reimDepartmentName` 精确匹配 `fk_reim_main.reim_department_name`。
4. `reimburserKeyword` 精确匹配 `fk_reim_main.reimburser_name` 或 `fk_reim_main.reimburser_no`。
5. `businessTypeName` 精确匹配 `fk_reim_main.business_type_name`。
6. `reimCompanyName` 查询主表缓存字段 `fk_reim_main.reim_company_names`。因为一个报销单可能有多个费用归属公司，实际实现时建议用包含匹配或按约定拆分匹配。
7. 默认按 `created_at DESC` 排序。

### 响应字段

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| total | Long | 总记录数 |
| pageNum | Integer | 当前页 |
| pageSize | Integer | 每页条数 |
| records | Array | 当前页数据 |

`records` 字段：

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| id | Long | 报销单主表 ID |
| reimNo | String | 报销单号 |
| billStatus | String | 单据状态编码 |
| billStatusName | String | 单据状态名称 |
| billTypeName | String | 单据类型名称 |
| reimburserName | String | 报销人姓名 |
| reimburserNo | String | 报销人工号 |
| reimDepartmentName | String | 报销部门名称 |
| reimCompanyNames | String | 费用归属公司名称；多个公司可用逗号拼接 |
| businessTypeName | String | 业务类型名称 |
| title | String | 报销标题 |
| reason | String | 报销事由 |
| allowanceAmount | BigDecimal | 补助金额 |
| createdAt | LocalDateTime | 创建时间 |
